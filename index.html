Boid Flocking

What is Boid Flocking?

Boid Flocking is an algorithm that was developed in 1986 by Craig Reynolds to simulate the flocking behavior of birds. The term "boid" is short for "bird-oid object".

How does it work?

Each boid follows three rules:

  1. Cohesion - the boid will try to occupy the center of the boids within it's visual range
  2. Separation - the boid will try to avoid colliding with the boids around it
  3. Alignment - the boid will try and match it's velocity (speed and direction) to the boids within it's visual range

The behavior of the flock can be modified by changing how much "force" is applied to the boids by each rule. For instance, if we were to lower the separation force, then the boids in the flock would fly closer together. If we were to turn the separation rule off completely, then the boids would not try to maintain distance from each other and would all overlap each other (you can try this by sliding the separation slider all the way to the left).

How is this implemented in my code?

Source code for my boid flocking implementation can be found on my GitHub.

A boid class is defined in a separate script (boid.js) which contains methods that control the behavior of each boid. These methods include the three rules mentioned above, as well as methods which control the boid behavior when they approach the edge of the canvas, calculate distance between a boid and another boid, and limit the speed of each boid so they dont fly too fast.

Another script (main.js) sets the size of the canvas, creates the boids and adds them to an array, and then runs the animation function which updates the positions of each boid and then schedules another animation frame which calls the animation function.

psuedocode provided by the following resource [1] was used to help implement the three rules, with some further guidance provided by the following boid flocking implementation [2].
